What is MXRoute Mailer?

MXRoute Mailer is a WordPress plugin that intercepts wp_mail() and sends email through MXRoute's HTTP API on port 443 instead of SMTP. This solves the common problem of hosting providers blocking outbound SMTP ports (25, 465, 587), which causes WordPress emails to silently fail on cloud platforms like Google Cloud and AWS.

Features

  • Smart-switch transport — API for lightweight emails, SMTP for attachments (creates sent folder copy for legal records)
  • Email queue — Background processing via WP-Cron with configurable batch size
  • File attachments — Full attachment support with persistent storage for temp files
  • Re-queue — Resend any failed or sent email directly from the logs page
  • Email logging — Full logs with search, filtering, and pagination
  • Reply-To support — Preserves form sender addresses as Reply-To headers
  • Test email tool — Verify your configuration with a single click
  • Automatic updates — Get new versions directly from GitHub
  • Works with any plugin or theme that uses wp_mail()

Installation

  1. Download the latest release from the Releases page.
  2. Upload and activate the plugin in WordPress.
  3. Go to Settings > MXRoute Mailer and enter your MXRoute server, username, and password.
  4. Send a test email to verify the configuration.

How it works

When a plugin or theme calls wp_mail(), MXRoute Mailer queues the email for background processing. WP-Cron then picks it up and applies the smart switch:

  • No attachments → Sent via MXRoute HTTP API (lightweight, no server storage)
  • Has attachments → Sent via SMTP through PHPMailer (creates a copy in your MXRoute sent folder for legal records and documentation)

The transport method is automatically selected and logged in the email details. You can re-queue any failed email from the logs page to try again.
